<p align="center"><sub><b>ABOUT</b></sub></p>

<h2 align="center">I care more about how systems work<br/>than what frameworks they're built on.</h2>

<p align="center">
I love building products — but more than that, I love understanding the machinery underneath them.<br/>
Instead of stopping at frameworks, I go looking for what they're standing on.
</p>

<br/>

> I'd rather write an HTTP server, a shell, and a memory allocator from scratch than take a socket, a process, or a heap for granted.

<br/>

<p align="center">
That instinct — to understand before I depend — shapes almost everything I build,<br/>
from production software used by real clinics to a tutor designed to work without the internet at all.
</p>
